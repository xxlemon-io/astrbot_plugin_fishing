### 🧧 完整红包系统

- **三种红包类型**
  - 普通红包：每个红包金额相同，人人有份
  - 拼手气红包：总金额随机分配，有人多有人少。
  - 口令红包：需要输入正确口令才能领取。

- **核心功能**
  - `/发红包 [金额] [数量] [类型] [口令]` - 发送红包
  - `/领红包 [口令]` - 领取红包（口令红包需要输入口令）
  - `/红包详情 [红包ID]` - 查看红包详细信息和领取记录
  - `/红包列表` - 查看当前群聊的所有活跃红包
  - `/撤回红包 [红包ID]` - 撤回自己发送的未领完红包
  - `/清理红包` - 管理员清理过期红包

- **智能机制**
  - 支持中文数字输入（如"一千"、"三个"）
  - 24小时自动过期机制，过期后自动退还剩余金额
  - 每小时自动清理过期红包，保持系统整洁
  - 群聊隔离：每个群的红包池独立，互不干扰
  - 防重复领取：每个用户每个红包只能领取一次

- **拼手气算法**
  - 使用改进的微信拼手气红包算法
  - 确保金额分配的随机性和公平性
  - 最小单个红包金额为总额的1%，避免过小金额
  - 采用正态分布，让红包大小呈现自然分布

- **安全与验证**
  - 完整的金额验证（最低100金币起发）
  - 余额检查，防止超额发送
  - 数量限制（最多200个红包）
  - 口令红包密码验证
  - 事务保证，确保金币扣除与红包创建的原子性

- **数据层实现**
  - 新增 `RedPacket` 数据模型（红包实体）
  - 新增 `RedPacketRecord` 数据模型（领取记录）
  - 新增 `SqliteRedPacketRepository` 仓储层
  - 新增 `RedPacketService` 业务逻辑层
  - 新增 `red_packet_handlers.py` 命令处理器

- **系统集成**
  - 完整的数据库表结构（`red_packets` 和 `red_packet_records`）
  - 自动化的后台清理任务
  - 与现有用户系统无缝集成
  - 完善的日志记录和错误处理

### 📝 使用示例

```
# 普通红包
/发红包 1000           → 发1个1000金币的普通红包
/发红包 1000 5         → 发5个各1000金币的普通红包
/发红包 十万 三个       → 发3个各10万金币的普通红包

# 拼手气红包
/发红包 1000 5 拼手气  → 发总额1000金币的拼手气红包，分5个

# 口令红包

# 领取红包
/领红包                → 领取普通/拼手气红包
/领红包 恭喜发财        → 领取口令红包

# 查看和管理
/红包列表              → 查看当前群的所有红包
/红包详情 1            → 查看红包详情
```
